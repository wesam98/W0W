## APKey HTB challenge

### Objective
![image.png](/img/A1.png)


### Solution

- Opened the APK in JADX and analyzed `MainActivity`. Found the login logic in the `onClick()` handler:

```java
if (MainActivity.this.f928c.getText().toString().equals("admin")) {
    // ... MD5 hash of password ...
    if (str.equals("a2a3d412e92d896134d9c9126d756f")) {
        makeText = Toast.makeText(applicationContext, b.a(g.a()), 1);
        makeText.show();
    }
}
```

- First checks if username equals `"admin"`
- Then computes MD5 of the entered password
- Compares the hash against a hardcoded value: `a2a3d412e92d896134d9c9126d756f`
- If match → shows the flag via `Toast`

- Instead of cracking the MD5 hash, the plan is to flip the condition so any password passes.
- To patch Smali, first needed to decompile APK, then navigate to the target Smali file and finally edit the onClick handler:

```bash
apktool d APKey.apk -o output/
cd output/smali/com/example/apkey/
nano 'MainActivity$a.smali'
```

- Located the hash comparison condition and flipped it: Original Check `if-eqz p1, :cond_1`, patched to `if-nez p1, :cond_1` — save and exit.
- Recompile APK: `apktool b output/ -o patched.apk`
- Sign new APK: `java -jar uber-apk-signer.jar --apks patched.apk --out signed/`
- Remember to uninstall original, install patched:

```bash
adb uninstall com.example.apkey
adb install signed/patched-aligned-signed.apk
```

### Result

- Launched the app → entered `admin` as username + any random password
- The condition now evaluates to `true` for any non-matching hash → flag displayed
