### Cat

#### Objective

- Analyze an Android Backup (`.ab`) file to extract sensitive data and retrieve a hidden flag.

---

#### What is an `.ab` File?

- `.ab` = **Android Backup** — Android's native backup format, generated via `adb backup`
- Internally it's a **ZLIB-compressed TAR archive** preceded by a plaintext header.
    
    ![image.png](/img/img1.png)
    
- Contains app data: `shared_prefs`, `databases`, `files`, and shared storage
- Android backups created without encryption (`adb backup` without `-ky`) expose **all app data and shared storage** to anyone with access to the `.ab` file — no root required.

---

#### Extracting The Backup

- Used [**android-backup-extractor**](https://github.com/nelenkov/android-backup-extractor) by [nelenkov](https://github.com/nelenkov) to unpack the `.ab` file:
    
    ```bash
    java -jar abe-540a57d.jar unpack cat.ab cat.tar
    tar -xvf cat.tar
    ```
    
- The extraction produced two top-level directories:
    - `apps/` → per-app backup data
    - `shared/0/` → device's shared storage (equivalent to `/sdcard/`)

---

### Investigating `apps/`

- Scanned all packages by size to identify non-system apps:
    
    ```bash
    du -sh ~/Downloads/apps/* | sort -rh | head -20
    ```
    
- All large packages were standard system apps (`com.android.*`, `com.google.*`) — not interesting.
- Two packages had `com.example` prefix, meaning they're **user-installed apps**, not system:
    - `com.example.android.notepad`
    - `com.example.android.rssreader`
- Investigated both:

```bash
find ~/Downloads/apps/com.example.android.notepad/ -type f
find ~/Downloads/apps/com.example.android.rssreader/ -type f
```

- Each only had a `_manifest` file — **no** `sp/`, `db/`, or `f/` directories, meaning the backup captured the app's existence but **no actual user data was backed up** for either app.
- So moved Shared folder, found only those 6 images viewed them and finally the flag was hidden inside one of them
    
    ![image.png](/img/img2.png)
    
- The flag is `HTB{ThisBackupIsUnprotected}`
